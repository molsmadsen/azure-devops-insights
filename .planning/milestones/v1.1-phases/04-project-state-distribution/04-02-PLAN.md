**update.mjs** — New script. Uses `child_process.execSync` and `fs`, `path`, `os` (all built-in). No config.mjs import needed (git operations don't need ADO credentials).

Arg parsing: `--check-config` (for SKILL.md Step 0 guard — but for update, config is not required; always return `{ configMissing: false }` so the skill proceeds).

Main flow:
1. Find plugin directory: use the same installed_plugins.json resolver pattern. Read `~/.claude/plugins/installed_plugins.json`, find entry matching 'adi', use its `installPath`. Fallback: use `process.env.CLAUDE_PLUGIN_ROOT` or the script's own directory (`path.resolve(dirname(fileURLToPath(import.meta.url)), '..')`).
2. Check for .git directory: run `git rev-parse --git-dir` in plugin dir. If fails: output `{ error: { type: 'no_git', message: 'Update requires git. Re-install via: git clone https://github.com/molsmadsen/azure-devops-insights' } }` and exit.
3. Get old HEAD: `git rev-parse HEAD`
4. Run `git pull` in plugin dir (capture output)
5. Get new HEAD: `git rev-parse HEAD`
6. If oldHead === newHead: output `{ updated: false, message: 'Already up to date.' }`
7. If different: get changelog via `git log --oneline ${oldHead}..${newHead}`, output `{ updated: true, oldHead: oldHead.slice(0,7), newHead: newHead.slice(0,7), changelog }`

All execSync calls: `{ cwd: pluginDir, encoding: 'utf8', stdio: 'pipe' }`. Wrap in try/catch for unexpected git errors.

JSON-only stdout. Progress to stderr.
