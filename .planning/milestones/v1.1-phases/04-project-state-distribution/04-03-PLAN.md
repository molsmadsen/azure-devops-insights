<task type="auto">
  <name>Task 1: Create summary.mjs sub-skill orchestrator</name>
  <files>scripts/summary.mjs</files>
  <action>
Create `scripts/summary.mjs` that orchestrates all four analysis sub-skills via child_process.

Arg parsing: `--days=<n>`, `--repo=<name>`, `--anonymous`, `--check-config`. Same pattern as other scripts.

Check-config mode: import configExists from config.mjs, return `{ configMissing: true/false }`.

Helper function `runSubSkill(scriptName, flags)`:
- Uses `execSync` to run `node <scriptPath> <flags>` where scriptPath is resolved relative to __dirname
- `{ encoding: 'utf8', timeout: 60000, stdio: ['pipe', 'pipe', 'pipe'] }`
- On success: parse stdout as JSON, return `{ ok: true, data: parsed }`
- On failure (catch): return `{ ok: false, error: e.message, scriptName }`
- Log progress to stderr: `process.stderr.write('Running ${scriptName}...\n')`

Main flow — run sub-skills SEQUENTIALLY (not parallel, for rate-limit safety):
1. `pr-metrics.mjs` with flags: `--days`, `--repo` (if provided)
2. `contributors.mjs` with flags: `--days`, `--repo`, `--anonymous` (if provided)
3. `bugs.mjs` with flags: `--days` (if provided) — note: bugs does NOT use --repo
4. `sprint.mjs` with no flags from summary (sprint uses --sprints which summary doesn't expose)

For each result: if `ok: false`, log warning to stderr but continue. If result `ok: true` but data contains `error` key (sub-skill returned an error JSON), treat as `{ ok: false, error: data.error.message }`.

Output JSON to stdout:
```json
{
  "prMetrics": { "ok": true, "data": {...} } | { "ok": false, "error": "..." },
  "contributors": { "ok": true, "data": {...} } | { "ok": false, "error": "..." },
  "bugs": { "ok": true, "data": {...} } | { "ok": false, "error": "..." },
  "sprint": { "ok": true, "data": {...} } | { "ok": false, "error": "..." },
  "meta": {
    "project": "<from config>",
    "generatedAt": "<ISO timestamp>",
    "skillsSucceeded": <count>,
    "skillsFailed": <count>
  }
}
```

If ALL four sub-skills fail: output `{ error: { type: 'all_failed', message: 'All analysis skills failed. Check your connection with /adi:setup.' } }`.
